5.3. API Versioning

To support updateable interfaces, we have chosen the following naming and inheritance scheme:

  • All service interfaces are descended from the empty tagging interface called com.quest.glue.api.services.Service.
  • All agent metadata will be specified in the agent's manifest file.
  • When we need to change an interface, we will create a new interface with a numeric suffix appended to it. For example, com.quest.glue.api.services.LogService would become com.quest.glue.api.services.LogService_0. However, we will only do this when we need to modify an existing interface. When we need to add new functionality, we will try to create an entirely new interface. For example, when we add a new property mechanism to replace com.quest.glue.api.services.AspService, we will create a new interface that could be called com.quest.glue.api.services.BetterPropertyService.
