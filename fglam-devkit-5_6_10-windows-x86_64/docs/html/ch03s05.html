3.5. UnitService

Associating units with metrics allows the server to properly scale and aggregate the metrics.

Each time a metric is collected, it should have a unit of measurement attached to it. The basic unit is 'count'. The UnitService provides an enumeration of all the basic units, and therefore can be used to construct more complicated units. For example: 

UnitService.getUnits("kilobyte/second")

An even more complex unit is used in the system time metrics on Linux, which are recorded in Jiffies. A Jiffie is a kernel-defined unit of time, which can be different depending on the kernel compilation: on stock 2.4, it is 10ms; on stock 2.5, it is 1ms; on some Red Hat kernels, it is 1/512s.

To represent these metrics, you would create units for them that look like the following: 

UnitService.getUnits("[10 millisecond]")
UnitService.getUnits("[1 millisecond]")
UnitService.getUnits("[0.001953125 second]")

The UnitService can be extended to support even more complex units than those presented here.
