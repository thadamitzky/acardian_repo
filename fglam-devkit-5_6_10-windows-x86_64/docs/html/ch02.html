2.1. Overview

The Foglight Agent Manager ships with an agent SDK that can be used to develop monitoring agents. This includes both an API used by agent developers to build new agents and a collection of tools for generating server artifacts, configuration files, helper classes, and agent packages that can be deployed into a running Foglight server.

When a Foglight Agent Manager Agent Archive (a GAR file) is deployed by Foglight into FglAM, it is scanned for a file named agent-manifest.xml. This file contains the definitions of all the agents contained within the Archive, including the name of the Java classes that implement agents, minimum FglAM versions to which the agents have been tested, version information about the agent package, and whether or not each agent is safe to run inside the FglAM process.

Each agent class mentioned in the manifest must implement one or more agent interfaces. FglAM uses these interfaces to control the lifecycle of the agent implementation without needing to know any specifics about the agent itself. In turn, a set of services is provided to the agent so that it can communicate with FglAM, other agents, or the Foglight Server.
