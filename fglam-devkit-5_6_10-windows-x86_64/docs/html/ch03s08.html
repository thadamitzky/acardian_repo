3.8. SupportBundleService

A support bundle service allows agents to contribute arbitrary information to a support bundle.

Support bundles are a collection of log files and state information often used by support and development teams for post-mortem debugging of problems in the field. Information can be added in two ways:

  • Through raw text, which is fed into a text file
  • In a list of files, which are copied into the support bundle

You can improve this process and cut down on the amount of time needed to diagnose and correct problems in the field by allowing an agent instance to submit information to the support bundle. By default, the Foglight Agent Manager collects the log files it maintains for all agent instances. Agent developers only need this service when information that is not in the log files would be useful in a support bundle. This may include, for example, runtime state information, or extra checks that are only performed for diagnostic purposes.
