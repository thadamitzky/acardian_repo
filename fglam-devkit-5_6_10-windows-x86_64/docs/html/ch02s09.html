2.9. Errors and Exceptions in Agents

If Agent.destroy() throws an exception, FglAM still considers the agent destroyed and deactivated.

If either Agent.startDataCollection() or Agent.stopDataCollection() throw an exception, FglAM considers the agent to be in a 'broken' state and attempts to de-initialize it by calling Agent.destroy().

If the agent is able to execute the Agent.startDataCollection() and Agent.stopDataCollection() methods successfully but later discovers an error, the agent should report the error to FglAM through the AgentStatusService.
