2.10. Services Available To Agents

When writing agents, there are a number of services provided by the FglAM host. These are are covered in detail in the Agent Services chapter. Here is a brief overview of the most commonly used services:

1. LogService

All logging done by an agent must use the LogService provided by FglAM. This ensures that all log messages are captured in a log file and that all log messages conform to the same format.

2. UnitService

The UnitService can be used for associating units with your data when it is submitted through the TopologyDataSubmissionService service.

3. TimestampService

All data sent to the server has its timestamps adjusted automatically by the TopologyDataSubmissionService to match those on the server. If an agent needs to adjust timestamps that are not handled by the TopologyDataSubmissionService, it can access the TimestampService to do it.

4. UpstreamConnectionService

The UpstreamConnectionService can be used to determine how often the Foglight Agent Manager will connect to its upstream and submit data, and that information can be used to scale an agent's data collection accordingly.

5. TopologyDataSubmissionService3

The TopologyDataSubmissionService3 is used to submit topology-based data samples associated with the current host, or a developer-specified host. It is also used to track the status of verified data submissions.
