2.11. Restrictions on Agents

Agents that run in process are subject to very strict security restrictions. In particular, they cannot load native code, write to the disk, execute processes, or create threads. All of these operations must be performed through the FglAM API services.

Agents that run out of process have no such restrictions, but are completely responsible for ensuring that their actions do not crash the VM hosting that agent, and all other agents from the same cartridge.
