Attribute Detail
allow-asynchronous-access
Flag used to control whether FglAM is allows to access agent code in an asynchronous manner or not. Asynchronous access means that more than one FglAM thread can access agents at any given time, wile non-asyncronous access means agnets will be protected by a lock that allows only single threaded access. This flag used to default to false, meaning unless explicitly allowed FglAM would only access agents using one thread at a time.
